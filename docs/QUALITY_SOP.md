# Linked3 AI 质检 SOP 标准作业程序

> **版本**: v16.0.19 | **适用**: Linked3 AI v16.0.12+ | **维护**: A部统筹中心
> **公理**: [信息熵减] 消除"怎么质检"的不确定性 + [系统降维] 1维文档替代N次口口相传

---

## 1. 目的与范围

### 1.1 目的
本 SOP 定义 Linked3 AI 插件从开发到发布的全流程质检标准，确保每个版本满足:
- **功能正确性**: 核心路径 100% 可用
- **安全性**: 无 XSS / SQL注入 / 路径穿越 / CSRF
- **兼容性**: PHP 7.4+ / WordPress 6.0+ / SQLite (Playground) / MySQL
- **性能**: 单页加载 < 3s, AJAX 响应 < 10s
- **可维护性**: 代码有注释, 变更有 CHANGELOG

### 1.2 适用范围
- 主版本发布 (如 v16.0.0 → v17.0.0)
- 小版本迭代 (如 v16.0.12 → v16.0.13)
- 紧急热修复 (Hotfix)
- 生态模块 (eco-*.php) 专项质检

## 3. 五阶段质检清单

### 阶段1: 静态代码扫描 (Static Analysis)

| 检查项 | 工具/方法 | 通过标准 | 失败处理 |
|--------|----------|---------|---------|
| PHP 语法 | `php -l <file>` | 0 error | 修复后重扫 |
| 编码规范 | PSR-12 人工抽查 | 关键文件合规 | 标记 TODO |
| 未使用变量 | IDE 静态分析 | 0 warning | 清理 |
| SQL 注入 | `grep -rn "\$wpdb->query.*\\\$_"` | 0 命中 | 改用 prepared statement |
| XSS | `grep -rn "echo \\\$_GET\|echo \\\$_POST"` | 0 命中 | 加 esc_html/esc_attr |
| 路径穿越 | `grep -rn "include.*\\\$_GET\|require.*\\\$_GET"` | 0 命中或白名单 | 加白名单 (见 v16.0.13) |
| nonce 校验 | `grep -rn "wp_verify_nonce"` 覆盖所有 AJAX | 100% 覆盖 | 补 nonce |

### 阶段2: 功能路径测试 (Functional Path)

| 模块 | 测试路径 | 预期结果 |
|------|---------|---------|
| 热词采集 | 生态→热词→选"全部源"→采集20个 | 返回20个去重热词, 持久化 |
| 长尾词生成 | 生态→长尾词→输入种子→生成 | 返回长尾词, 自动保存, 显示使用状态徽章 |
| 一键生态 | 生态→一键生产→输入主题→执行 | 5阶段进度条推进, 生成文章+图片 |
| CSV批量 | 长尾词库→导出CSV→批量生成 | 每词生成1篇, 标记已用 |
| 路由白名单 | URL `?eco_sub=../../../etc/passwd` | 回退到 synergy, 不报错 |

### 阶段3: 边界与异常测试 (Edge Case)

| 场景 | 输入 | 预期 |
|------|------|------|
| 空输入 | 主题为空 | 提示"请输入主题", 不发请求 |
| 超长输入 | 主题 10000 字 | 截断到 200 字 + 提示 |
| 特殊字符 | 主题含 `<script>` | 转义存储, 不执行 |
| AI 超时 | 模拟 AI 30s 无响应 | 显示错误四元组 (阶段3) |
| API Key 缺失 | 未配置 Key | 错误四元组 hint 指向设置页 |
| nonce 过期 | 12h 后操作 | 错误四元组 (阶段2) 提示重新登录 |
| 并发保存 | 500 词同时保存 | array_slice 限流, 不崩溃 |

### 阶段4: 兼容性测试 (Compatibility)

| 环境 | 测试点 | 标准 |
|------|--------|------|
| PHP 7.4 | 无 named args / enum | 语法兼容 |
| PHP 8.2 | deprecation warning | 0 warning |
| WordPress 6.0 | REST API / Hook | 正常加载 |
| WordPress 6.5 | Block editor | 无冲突 |
| MySQL 5.7 | utf8mb4 | 正常读写 |
| SQLite (Playground) | option 表 | update_option 正常 |
| 移动端 375px | 后台布局 | Grid 自适应, 无溢出 |
| 暗色模式 | 卡片/输入框 | 可读, 无对比度问题 |

### 阶段5: 性能与回归 (Performance & Regression)

| 指标 | 测量方法 | 阈值 |
|------|---------|------|
| 后台首屏 | DevTools Network | < 3s |
| AJAX 热词采集 | console.time | < 8s (6源串行) |
| AJAX 一键生态 | console.time | < 60s |
| 内存峰值 | memory_get_peak_usage | < 128MB |
| DB 查询数 | SAVEQUERIES | < 50/页 |
| CSS 体积 | gzip 后 | < 50KB |

## 5. 发布检查清单 (Release Checklist)

- [ ] `php -l` 全文件 0 error
- [ ] CHANGELOG.md 已更新本次版本
- [ ] 主文件版本号已递增 (`LINKED3_VERSION`)
- [ ] 五阶段质检全部通过
- [ ] 错误码字典已同步 (如有新错误)
- [ ] 移动端 375px 视觉验收
- [ ] SQLite (Playground) 兼容验证
- [ ] 无硬编码 API Key / 密码
- [ ] 新增 option 有默认值兜底
- [ ] 新增 AJAX 有 nonce 校验 + 权限校验

## 6. 回退条件 (Rollback Triggers)

出现以下任一情况, 立即回退到上一稳定版:

1. 发布后 1 小时内收到 ≥ 3 个致命 Bug 报告
2. 核心路径 (热词采集/长尾词生成/一键生态) 不可用
3. 白屏 (WSOD) 影响后台加载
4. 数据库写入异常导致 option 损坏
5. 安全漏洞 (XSS/SQL注入/路径穿越) 被披露

**回退操作**:
```bash
# 1. 禁用当前版本
wp plugin deactivate linked3-ai
# 2. 替换为上一稳定版 zip
wp plugin install linked3-ai-v16.0.12.zip --force
# 3. 启用
wp plugin activate linked3-ai
# 4. 验证
wp option get linked3_version
```
